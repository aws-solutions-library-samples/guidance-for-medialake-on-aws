# No Emojis in Code Comments

RULE: Do _not_ use emojis (e.g. 😀, 🚀, ✅) in any code comments.

- Use clear, professional text (e.g. “Fixes edge case where input is null”).
- Emojis add noise and can cause encoding issues or reduce readability.

RULE: Remove any test scripts that you created to temporarily test the implementation
RULE: Put all documentation you produce in the INTERNAL-ai-documentation folder at the root of the workspace folder, name it based off the feature or technology that it's for
